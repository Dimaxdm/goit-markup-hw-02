# Homework. Module 1. HTML Fundamentals

1. Create a repository named `goit-markup-hw-01`.
2. Complete the HTML markup for the ["Homework #1"](https://www.figma.com/design/folOvI69KUmwYN47Njpr0O/Web-Studio--Version-5.1-?node-id=0-1&p=f&t=gMWcWNxgacyJYomK-0) page layout without any CSS styling.
3. Set up GitHub Pages and add a link to the live page in the About section of the GitHub repository.
4. In the `goit-markup-hw-01` folder, create the project structure as shown in the diagram below.

## Project Requirements

* [x] - The HTML file is named `index.html`.
* [x] - The project root contains an `images` folder with all image assets.
* [x] - File names do not contain Cyrillic characters or spaces. Names use only English letters, numbers, and words.
* [x] - The source code is formatted using Prettier.
* [x] - All images and text content are taken from the provided design mockup.
* [x] - The code follows the provided coding guidelines.

## Markup

* [x] - All markup is written in the `index.html` file.
* [x] - HTML markup is implemented for all elements in the design mockup.
* [x] - Tags are used according to their semantic meaning.
* [x] - The HTML passes validation without errors.
* [x] - The root `<html>` element includes a `lang` attribute with the appropriate language value.
* [x] - The `<head>` section specifies UTF-8 character encoding.
* [x] - The page structure includes the semantic tags `<header>`, `<main>`, and `<footer>`.

## Header
* [x] - The header has the following ID:

```html
<header id="header">
```

* [x] - Navigation items in the header are anchor links pointing to the corresponding sections of the page.
* [x] - The logo in the header is a text link, not an image.
* [x] - The logo link points to `./index.html`.
* [x] - The logo is not part of the `<nav>` element.
* [x] - The `<nav>` element is used only once on the page, inside the header.
* [x] - Contact information in the header (email and phone number) is located outside the `<nav>` element and wrapped in an `<address>` tag.
* [x] - Email and phone links must be functional, opening the user's email client or calling application when clicked.

## Hero Section
* [x] - The section has the following ID:

```html
<section id="hero">
```

* [x] - The "Order Service" element is a `button` with `type="button"`.

## Features Section
* [x] - The section has the following ID:

```html
<section id="features">
```

* [x] - All section headings, except for the Hero section, are marked up using `<h2>` tags.
* [x] - The "Our Features" section must include an `<h2>` heading with the text "Our Features". In future assignments, this heading will be visually hidden, but for this assignment it should remain visible.
* [x] - Throughout the layout, groups of similar elements are organized using unordered lists (`<ul>`).

## Team Section
* [x] The section has the following ID:

```html
<section id="team">
```

* [x] - All `<img>` elements include size attributes, at minimum the `width` attribute.
* [x] - All `<img>` elements include an `alt` attribute containing a brief description of the image content. The description must be written in the same language as the rest of the page content.
* [x] - Images are exported from the design mockup in JPG format.

## Footer
* [x] - The footer has the following ID:

```html
<footer id="footer">
```

* [x] - The logo in the footer is a text link, not an image.
* [x] - The logo link points to `./index.html`.
